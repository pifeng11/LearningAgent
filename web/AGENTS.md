# Web Frontend Instructions

## Communication
- 迭代本目录代码前，必须先用中文说明计划并等待用户确认。
- 需求不明确时先提问，不要自行扩大功能范围。

## Product Direction
- 前端定位是 Learning Agent 的实际工作台，不做营销落地页。
- 默认优先打磨聊天、学习计划、练习、复盘等高频学习流程。
- 任何新增页面都应能直接完成一个用户任务，避免只展示说明文字。

## Design
- 使用 Vite、React、TypeScript、Tailwind CSS 和 lucide-react。
- 保持界面简洁、专业、信息密度适中，适合长期学习和反复使用。
- 不使用单一色相铺满全站，避免大面积紫色、深蓝、米色或棕橙主题。
- 不使用装饰性渐变球、光斑、无意义插画或嵌套卡片。
- 按钮、输入框、工具栏、状态区必须在移动端和桌面端都不重叠、不溢出。
- 操作按钮优先使用 lucide 图标搭配简短文字，必要时补充 `title`。
- 文本字号不要随视口宽度缩放，字距保持默认。

## Engineering
- 只修改与当前任务直接相关的文件。
- 保持组件小而明确，优先复用现有类型和 API 客户端。
- 新增接口调用时，优先通过 `src/lib/api.ts` 统一维护。
- 不把密钥、真实用户信息、个人身份信息写进代码或示例数据。
- 注释使用中文，且只在解释意图或复杂逻辑时添加。

## API Integration
- 默认使用 `/api/v1/agent/chat/stream` 的 SSE 流式接口。
- 开发环境通过 Vite proxy 访问 Go 服务，避免在组件里硬编码后端域名。
- 请求字段保持与 Go 后端一致：`user_id`、`session_id`、`message`。
- 必须处理加载中、流式增量、完成和错误状态。

## Verification
- 修改后至少运行 `npm run build`。
- 如果改动了样式或交互，应尽量启动本地页面做人工检查。
- 不要声称测试或构建通过，除非实际运行过。
